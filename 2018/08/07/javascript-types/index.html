JavaScript 数据类型笔记

这篇文章简单总结了JavaScript中存在的数据类型及常用判断和处理方法，以及一些相关面试题。

概述

JavaScript是一种弱类型语言，也叫动态语言，所有的变量都通过var(/let/const in ES6)声明，而变量类型在运行过程中自动确定。
不同类型间转换的界限也不严格，同名变量重新赋值就可以成为另一个类型。

JavaScript有六种原始数据类型，为方便记忆，从简单到复杂，分别是：
null, undefined, boolean, number, string, symbol(ES6)
以及一种特殊的数据类型：
object
我们常用到的array是object的一种。

把变量作为判断条件时，比如作为if…else…等语句的条件时，会对输入进行Boolean(x)转化，转化条件如下
（1）空数据类型null和undefined，以及空字符串“” 对应false

Number

在JavaScript中，浮点数与整数都属于number类型。要注意的是，浮点数在JavaScript中不能做精确运算。
此外，在这一数据类型下有几个重点。

NaN(Not-a-Number)

如果用任意数字除以0，在JavaScript中不会报错，而是返回NaN。NaN做任何运算都会返回NaN。
NaN非常任性，有一个在所有数值中独一无二的特性：

但NaN的数据类型是number，只是数值是NaN。为了加深理解，这里引用一下书中定义：

NaN literally stands for “not a number”, though this label / description is very poor and misleading. It would be much more accurate to think of NaN as being “invalid number,” “failed number,” or even “bad number,” than to think of it as “not a number.”
—— < You Don’t Know JS>

  • isNaN()就是在对NaN的误解下产生的函数。用这个函数测试NaN与其他所有非number类型的值都会返回true。

在ES6之后，为了弥补这个问题，引入了Number.isNaN()。
该函数可以正确判断NaN，但不兼容某些老版的浏览器。所以建议在使用前判断浏览器是否支持，并在不支持的情况下对该函数进行定义。

String

String大致与array近似，可以通过str[n]来访问某一个char，但是string中的char不能被单独改变。
也就是说，string可以被重新赋值，但是不能以str[n] = “x”的方式来部分改写。
